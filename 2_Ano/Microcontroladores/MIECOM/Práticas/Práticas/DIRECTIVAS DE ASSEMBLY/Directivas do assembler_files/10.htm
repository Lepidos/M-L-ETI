Directivas do assembler

 

 

Existem oito tipos de directivas:

 

• Para controlo de segmentos.

 

• Para definição de símbolos.

 

• Para inicialização de memória.

 

• Para reserva de memória.

 

• Para ligação do programa.

 

• Para controlo de endereços.

 

• Para definição de macros.

 

• Outras directivas.

 

 

De seguida serão apresentadas as directivas correspondentes a cada tipo.

 

Directivas para controlo de segmentos

 

Designa-se por segmento um bloco de memória de código ou de dados criado pelo assembler.

 

As directivas para controlo de segmentos servem para criar e seleccionar segmentos. Estão definidas cinco classes de memória que podem ser utilizadas na definição ou criação do segmento:

 

• BIT – bit em memória interna. Endereços até de 0 a 255.

 

• CODE – memória de código. Endereços de 0000H a 0FFFFH.

 

• DATA – memória de dados acessíveis através de endereçamento directo. Endereços de 0 a 255 (posições genéricas de 0 a 127 e SFRs de 128 a 255).

 

• IDATA – memória de dados interna. Endereços de 0 a 255.

 

• XDATA – memória de dados externa. Endereços de 0000H a 0FFFFH.

 

As directivas deste tipo são apresentadas na tabela 6.

 

 

 

Tabela 6 - Directivas para controlo de segmentos

 

 

Exemplos:

 

mprogr SEGMENT CODE ; define o segmento de código mprogr .

 

RSEG mprogr ; activa o segmento mprogr.

 

dd1 SEGMENT DATA ; define o segmento de dados dd1.

 

mprog1 SEGMENT CODE AT 1000H ;define o segmento de código mprogr1 com início no endereço 1000H.

 

xdd SEGMENT XDATA PAGE ; define o segmento de dados externos xdd alinhado com uma página de 256 bytes.

 

BSEG AT 20H ; define um segmento absoluto de bit com início no endereço 20H.

 

 

Directivas para definição de símbolos

 

As directivas para atribuição de símbolos atribuem determinado valor a um símbolo.

 

As diferenças entre as várias directivas deste tipo relacionam-se com o tipo de valor a atribuir ao símbolo.

 

A lista das directivas deste tipo é apresentada na tabela 7.

 

 

 

Tabela 7 - Directivas para definição de símbolos

 

 

Exemplos:

pt_leitura SET R1 ;utilizar R1 como pt_leitura.

falso EQU 0 ; utilizar falso para representar 0.

flag1 BIT 50H ; utilizar o bit cujo endereço é 50H como flag1.

restart CODE 00H ; faz restart igual à memória de código 00H.

PORT1 DATA 90H ; atribui a PORT1 o valor 90H correspondente ao endereço do registo associado ao porto P1.

Directivas para inicialização de memória

As directivas para inicialização de memória permitem inicializar em memória de código posições de 8 ou 16 bits.

A tabela 8 apresenta as directivas disponíveis com esta função.

 

 

 

Tabela 8 - Directivas para inicialização de memória

 

 

Exemplos:

 

segmento1: DB 7EH,60H,6DH,79H ; inicializa 4 posições de 8 bits, com início na posição cujo endereço corresponde à etiqueta segmento1, com os valores indicados.

 

prompt: DB ‘Ola, como esta!’ ; neste caso a inicialização é feita com os códigos ASCII correspondentes aos caracteres entre ‘’.

 

tab_salto: DW end1, end2, end3 ; inicializa o segmento de código com os endereços correspondentes às etiquetas end1, end2 e end3.

 

 

Directivas para reserva de memória

 

Este tipo de directivas serve para reservar espaços de memória de 1 ou de 8 bits. As directivas deste tipo são apresentadas na tabela 9.

 

 

 

Tabela 9 - Directivas para reserva de memória

 

 

Exemplos:

 

DBIT 9 ; reserva nove bits.

 

DS 25 ; reserva 25 bytes no segmento activo.

 

 

Directivas para ligação do programa

 

As directivas para ligação do programa utilizam-se quando o programa a desenvolver se encontra distribuído por vários módulos.

 

Como os módulos necessitam de partilhar informação torna-se necessário identificar a origem da informação (quando externa) ou qual a informação que vai ser partilhada.

 

As directivas deste tipo utilizam-se para essas funções e encontram-se na tabela 10.

 

 

 

Tabela 10 - Directivas para ligação do programa

 

 

Exemplos:

 

PUBLIC s1, s2, s3 ; torna públicos os símbolos s1, s2 e s3.

 

EXTRN CODE(s1, s2) ; declara que os símbolos s1 e s2 estão definidos noutro módulo.

 

NAME módulo1 ; especifica que o módulo objecto a se deve chamar módulo1.

 

 

Directivas para controlo de endereços

 

Estas directivas servem para determinar o endereço em que o código deve ser posicionado e para selecção do banco de registos a utilizar.

 

Estas directivas encontram-se na tabela 11.

 

 

 

Tabela 11 - Directivas para controlo de endereços

 

 

Exemplos:

 

ORG 2000H ; posiciona o contador de localização no endereço 2000H

ORG reset ; em que reset é um símbolo predefinido correspondendo a um dado endereço.

 

USING 2 ; selecciona o banco de registo número 2.

 

 

Directivas para definição de macros

 

Este tipo de directivas permite definir macros. Designa-se por macro um conjunto de instruções reunidas sob uma determinada etiqueta utilizando directivas específicas.

 

Quando a etiqueta é referida no programa o assembler procede à sua substituição pelo código referido anteriormente.

 

Nem todos os assemblers suportam a utilização de macros. Estas directivas são apresentadas na tabela 12.

 

 

 

Tabela 12 - Directivas para definição de macros

 

 

Uma macro define-se do seguinte modo:

 

nome_macro: MACRO ard1, arg2, ... ; comentário

.

.

ENDM
